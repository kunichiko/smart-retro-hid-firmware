# smart-retro-hid-firmware 設計メモ

## 概要

スマートフォン（Flutter アプリ）と USB-MIDI で接続し、レトロ PC の HID デバイス（キーボード・マウス・ジョイスティック）を模倣するマイコンファームウェア。

## マイコン

**CH32X035F8U6** (WCH, TSSOP20)

| 項目 | 仕様 |
|------|------|
| コア | QingKe RISC-V4C (RV32IMAC), 48 MHz |
| Flash | 62 KB |
| RAM | 20 KB |
| USB | USB 2.0 Full-Speed Device + USB PD PHY |
| VDD | 2.0 - 5.5 V（5V 駆動でレトロ PC と直結可能） |
| パッケージ | TSSOP20 |
| 単価 | 約 $0.27 |

### 選定理由

- 5V ネイティブ動作 → レベルシフタ不要でレトロ PC（TTL 5V）と直結
- USB 内蔵 → USB-MIDI デバイスとして動作可能（ch32v003fun fsusb ドライバ使用）
- TSSOP20 → 小型ドングルに最適、手はんだ可能
- USB PD PHY 内蔵 → USB-C 接続時のネゴシエーションに有利
- 安価 → 量産時のコスト低減

## 対応デバイス

### 1. ATARI 仕様ジョイスティック

| 項目 | 仕様 |
|------|------|
| 接続 | D-SUB 9 ピン |
| 信号方式 | パラレル I/O（各方向 + ボタンが独立したピン） |
| 電圧 | TTL 5V |
| ピン数 | 最大 6 本（上下左右 + ボタン 1-2） |

信号線は単純な GPIO 出力で実装可能。

### 2. X68000 キーボード

| 項目 | 仕様 |
|------|------|
| 接続 | DIN 7 ピン |
| 信号方式 | 非同期シリアル (UART) |
| ボーレート | 2400 bps |
| フォーマット | 8N1 (8bit, パリティなし, ストップビット 1) |
| 電圧 | TTL 5V |
| 方向 | 双方向 |

#### キーボード → X68000 本体（キーコード送信）
```
bit7: 0 = 押下 (make), 1 = 解放 (break)
bit6-0: キーコード (0x00-0x7F)
```

#### X68000 本体 → キーボード（LED 制御等）
LED 制御コマンドにより、かな・ローマ字・CAPS・INS 等の LED を制御。
ファームウェアは受信した LED 状態を USB-MIDI 経由でスマホアプリに通知する。

## USB-MIDI プロトコル設計方針

USB-MIDI の SysEx (System Exclusive) メッセージを利用して、HID 制御データをカプセル化する。

- スマホ → マイコン: キー押下/解放、ジョイスティック方向・ボタン
- マイコン → スマホ: LED 状態変化、デバイス接続状態

詳細は [smart-retro-hid-protocol](https://github.com/kunichiko/smart-retro-hid-protocol) リポジトリで定義。

## ピンアサイン (CH32X035F8U6 TSSOP20)

```
          CH32X035F8U6
          ┌────────────┐
    VDD ──┤ 1       20 ├── VSS (GND)
    PA4 ──┤ 2       19 ├── PA3
    PA5 ──┤ 3       18 ├── PA2
    PA6 ──┤ 4       17 ├── PA1
    PA7 ──┤ 5       16 ├── PA0
  USB_DM ─┤ 6       15 ├── PB0
  USB_DP ─┤ 7       14 ├── PB1
    PC0 ──┤ 8       13 ├── PB3
    PC2 ──┤ 9       12 ├── PB4
    PC3 ──┤ 10      11 ├── PC1 (UART TX / SWDIO)
          └────────────┘
```

### ピン割り当て案

| ピン | 機能 | 用途 |
|------|------|------|
| Pin 6,7 | USB_DM, USB_DP | USB-C 接続（スマホ） |
| Pin 11 (PC1) | USART1 TX | X68000 キーボード: キーコード送信 |
| Pin 8 (PC0) | USART1 RX | X68000 キーボード: LED コマンド受信 |
| Pin 16 (PA0) | GPIO OUT | ATARI ジョイスティック: 上 |
| Pin 17 (PA1) | GPIO OUT | ATARI ジョイスティック: 下 |
| Pin 18 (PA2) | GPIO OUT | ATARI ジョイスティック: 左 |
| Pin 19 (PA3) | GPIO OUT | ATARI ジョイスティック: 右 |
| Pin 2 (PA4) | GPIO OUT | ATARI ジョイスティック: ボタン 1 |
| Pin 3 (PA5) | GPIO OUT | ATARI ジョイスティック: ボタン 2 |

※ PC1 は SWDIO と共有。デバッグ時は UART を切り離す必要あり。
※ アダプタの種類（ジョイスティック / キーボード）ごとに基板を分けるため、
  全ピンを同時に使用する必要はない。

| 書き込み | minichlink |

### 備考: TinyUSB について

TinyUSB は CH32X035 を直接サポートしていない（CH32V20x/30x のみ）。
ch32v003fun の fsusb ドライバを使用して USB-MIDI デバイスを実装している。

## メモリ使用量（実測値: 2026-05-01）

```
Flash:  4,988 / 63,488 bytes ( 7.9%)
RAM:      812 / 20,480 bytes ( 4.0%)
```

実装済みモジュール:
- ch32v003fun コア + USB-MIDI ドライバ (fsusb)
- SysEx ネゴシエーション (Identify, Capability, Reset)
- X68000 キーボード UART 制御
- ATARI ジョイスティック GPIO 制御
- MIDI メッセージ → レトロ PC 信号変換

→ Flash/RAM ともに十分な余裕あり。
